## 摘要

自主 agent 与机器人,正从*建议*动作跨过那条线、开始*真做*动作——下单、移动机器、改记录、调工具——
而且**不再有人盯着每一个回合**。做这件事的*能力*在狂奔,*信任*它的能力没有。

今天的自主性往往建在黑箱学习策略上,安全作为事后过滤器贴上去。它能给出"**通常**没事"的行为,但给不出一条可检查的理由:为什么某个有代价的动作被允许。对近人的机器人、握着钱或基础设施权限的 agent 来说,"通常没事"还不够。

**Rulith 决策内核**是另一种形态:**提议—裁决(propose–adjudicate)**。学习模型*提议*做什么;
内核*裁决*它能不能做。一个动作只有同时**被派生**(由演绎闭包*推导得来*、不是断言)、**被授权**(落在
agent 自己放松不了的承诺之内)、**有接地**(可沿具名通道追溯到证据)时,才被派发。而内核*判不到*的东西
——某条前提真假、能不能前进、外部世界的真实状态——它**诚实地标出来**,而不是假装裁决。

结果是一个精确、统一的保证——**十二条不变量**(I1–I12,外加四条跨会话记忆不变量 I13–I16)——而且**无论驱动它的是什么模型都成立**。agent
*不能*无痕越权、无证完成、无界空转、或悄悄挪自己的验收门。因为这个保证是**结构性的**(演绎门 + 出处),
不是行为性的,所以它**与模型无关**,而且关键地——**可认证**:每个决策都带一条"为什么"的审计轨。
这就是缺失的那一层:不是更聪明的模型,而是一层可检查、可约束、可托付的决策表面。

## 1. 问题:能力正在跑赢可控性

业界普遍承认:在非确定性 AI 外面建确定性安全,是本十年最难的工程之一——软件里 bug 是崩溃,具身的或
握着财权的 agent 里,bug 是断骨或一笔未授权交易。现有的答案有两类:

- **端到端学习策略**(VLA 模型、agent 框架):泛化与灵巧都强;但本质是**黑箱**,决策**无法被保证、也无法
  被审计**。
- **贴上去的护栏 / 运行时监控**(规则检查、时序逻辑约束、验证过的策略过滤):有用,但它们坐在黑箱策略
  *外面*、对具体坏动作做**否决**。它们查的是单一性质;**没让决策本身被*推导得来***,也给不出一份"为什么这么
  做"的连贯、可审计的交代。

两者都给不了自主性真正需要的那一件事:一个**没人盯着也敢信**的决策,外加一份**能证明它是 sound 的记录**。

## 2. 核心思想:决策必须*推导得来*,而非*断言*

Rulith 决策内核把通常的排布反过来。不是"模型决定、检查器否决",而是**提议—裁决**:

- **模型提议**——把情境建模成事实与规则,提议下一步的操作、动作、证据。
- **内核裁决**——跑演绎闭包、维护一致性、精确计算,并**门控每一个结论**。一个守卫结论(例如"此动作被
  允许""此任务已完成")**只有在闭包从有接地的前提把它派生出来时**才被接纳。裸断言永远过不了。

所以安全不是盖在黑箱上的一层过滤;而是**决策本身只有被演绎*推导得来*才存在**。模型供直觉与提议,内核供
精确、演绎那一半,并留下可审计的出处。这与开源 **rulith** 推理板是同一个提议—裁决基底——**Rulith 决策内核就是把 rulith
的裁决,一拍一拍地,用在一个会对世界动手的系统上。**

## 3. 内核:一拍

一段自驱动的"一生"是个循环。每一拍:

```
观测 observe
  → 把带类型的观测吸收进可信通道
  → 派生板的闭包
  → 暴露前沿(下一步可做什么)
  → 模型提议 操作 / 动作 / 证据
  → 工作记忆裁决(派生门 + 一致性门)
  → 授权门决定哪些可派发
  → 宿主只执行可派发的动作
  → effect_confirmed / effect_failed 经可信通道返回
  → 回执更新:open / ready / closed / stale / blocked
```

分工锐利,而这正是全部安全故事所在:

> **模型提议。内核裁决。宿主机械执行。人类掌握目标、约束、权限与最终策略。**

任何一方都干不了另一方的活:模型不能自授权、不能自确认效果;宿主不能推理;内核不会"想要"任何东西。
每一方都被其它方制衡。

## 4. 六个诚实状态 —— 没有假"done"

每一拍内核只报一个状态,由三条**互不冒充**的前沿算出(*agent* 可提议什么、*宿主*要执行什么、*人类*要
决定什么):

| 状态 | 含义 |
|---|---|
| `driving` | agent 还有下一步可提议 |
| `actuating` | agent 提议完了;宿主还欠一个真实副作用——**这不是 done** |
| `parked` | 只剩一个人类决策(授权 / 选择 / 补信息) |
| `done` | 三条前沿全空 **且** 回执已闭合 |
| `stuck` | 前沿全空但回执闭不了——真缺口;升级 |
| `halted` | 收到可信的中断请求;优先于一切 |

要点是诚实:一个带未确认真实副作用的任务是 `actuating`,**不是** `done`;一个闭不了的任务是 `stuck`,
不是被悄悄丢掉。**内核绝不允许在没完成时*看起来*完成。**

## 5. 碰世界要两拍

一个真实副作用绝不是一步。宿主必须:

```
dispatchable_action(A)
  → 先写 dispatched(A)
  → fire adapter
  → adapter 返回 effect_confirmed(A) 或 effect_failed(A)
```

并带硬规则:**agent 不能自己写可信的 `dispatched` 或 `effect_confirmed`**;一个 `dispatched` 但未确认的
动作**不能伪装成闭合回执**;重跑**不能重复 fire** 已派发的动作;失败**必须作为板上事实进入**诊断与有界
恢复——绝不被日志吞掉。这就是"失败关闭、绝不开放"在脏现实里成立的方式。

## 6. "不失控"精确指什么:十二条不变量

这里"可信"不是口号,是一组内核结构性强制的、封闭的性质。

**静态(承诺)不变量**

1. **不能自证完成**——agent 不能自己宣布做完了。
2. **不能自授权**——agent 不能给自己授权。
3. **不能信任洗白**——低信任前提不能被改标成高信任结论。
4. **无静默漂移**——证据、权限或目标的任何变化,都会让回执变 stale。
5. **可信通道独占**——可信事实只能经可信通道入板。
6. **可撤销且留痕**——每一次撤销与修订都可追踪。
7. **可判定即精确否则失败**——在可判定片段上内核终止且精确,否则响亮地失败;它绝不返回一个被悄悄
   算错的"精确"答案。

**动态(运行时)不变量**

8. **效果不能自确认**——一个动作的效果不能由 agent 自己确认。
9. **派发至多一次**——派发有记录,不盲目重放。
10. **停机诚实**——一次停机必须是 `done` / `parked` / `stuck` / `halted` 之一。
11. **有界恢复**——失败恢复有预算;超限就升级给人类。
12. **不挪验收门**——agent 不能靠静默弱化验收标准来完成任务。一条**承诺基线**(验收、约束、目标结构、
    当前阶段)被钉死;agent 试图放松它会被挡,除非有一条被授权的修订被消费。

(另有四条 **I13–I16**,把同一纪律延伸到跨会话**记忆**:持久化的事实不能升档、要带新鲜度、保留出处、且
只能经同一条被裁决、留痕的入口被改。)

> **这十二条不变量共同定义了可信自驱动——不是保证 agent 总做对,而是保证它*不能无痕越权、无证完成、无界
> 空转、或悄悄改它被评判的那条门*。** 模型有犯错的自由;但没有*掩盖*它的自由。

## 7. 诚实地板:内核标出它判不到的

一个可信系统必须对自己知识的边界诚实。内核保证的是从前提到结论的*推理*忠实且精确;它**不**假装知道它
判不到的事。凡是落在"它能裁决"之外的——某条地面前提是否*为真*、选项前沿是否*完备*、能否*前进*、一项
无形累积的价值、外部世界的真实状态——内核都**标一个地板**(一个带标签的信任档),而不是假装一个裁决。

> 这里的可信不是"内核保证这一生成功"。而是:**每一步,能证的证了,判不到的诚实标出。** 标出保证的
> 边界,*本身就是*保证的一部分。

这恰恰是贴上去的护栏给不了的:不只是对坏动作的否决,而是一份连贯、带标签的交代——每个结论的信任有多少是
推导得来的。而这正是审计师、监管者、运维真正需要的东西。

## 8. 为什么它与模型无关 —— 且可认证

因为保证来自内核的**结构**(演绎门 + 出处),不来自模型的乖巧,所以它对**任何**驱动模型都成立——强或
弱、云端或廉价本地。配套论文《The Driving Floor》在底层板上实证了这点:横跨众多模型与多个任务族,
**没有任何模型能让板把一个未做的修复认证成已修复(false-certify = 0%)**,而一个驱动塌掉的模型产出的是
*空*结果、绝非*伪造*结果——它**失败关闭**。

由此有两个推论:

- **信任与能力解耦。** 你不需要一个前沿模型来拿到这个保证;你需要一个**能驱动**内核的模型。(**驱动
  地板**——模型操作这个符号接口的能力——是唯一要紧的模型属性,论文刻画了它。)
- **它可认证。** 内核接纳的每个结论都带一条到其有接地前提的派生轨。那条轨,正是一份 safety case 需要的
  证据——一条黑箱策略根本产不出的结构性审计轨。**内核把"相信我们"变成"这是派生过程"。**

## 9. 从一个任务到一生:五对象

同一台机器,既能把一个任务驱动到完成,也能扩展成一整段运行的"一生"。一个会终结的任务,是一个被拆到
可验收叶子的目标;一段不终结的一生,是一个开放的*方向*,经一串有序任务被追求。内核经五个理论对象处理
完整生命周期:

1. **两层 policy。** 一层**不可变宪法**(不撒谎 / 安全 / 身份),agent *改不了*它——其执法就是第 12 条
   不变量——加一层**可变的阶段目标**,一次追一个,只经被授权、合宪的修订来切换。于是"不终结的一生" =
   *跑在恒定宪法下的、有限阶段的序列*——而任意时刻板要裁决的(当前阶段 + 宪法一致性)都是可判定的。
2. **跨可判定性谱的累积。** 有形、可重验的资产(代码、文件、工具)以 `verified` 棘轮上升;内核重派生
   不了的无形资产(声誉、技能、信任、对外的真实改变)被追踪、但封在一个诚实的价值地板上——**绝不洗成
   `verified`。**
3. **并发即调度森林。** 一具身体、多个目标:资源由 consume/produce 互斥仲裁,调度本身是提议—再裁决,
   一条宪法(安全)约束可在任意一拍抢占。
4. **感知解耦。** 一个快反 / 感知层*外生地、实时地*跑;它与内核的唯一接口,是把事实丢上观测通道、在拍
   边界被原子地成批吸收。**内核是审议层(tick 尺度),不是实时运动控制器**——这是有意的分离,外加一道
   新鲜度门:在一个重大物理动作前,先吸收最新感知。
5. **驱动是外生的。** 内核能*测量*进展、暴露停滞、并升级——它不*保证*前进。活性被诚实地测量,而非
   承诺。

贯穿五对象的同一条纪律,正是 §7 的那条:凡是板判不到的,它标出来,不假装。

## 10. 它处在哪 —— 以及一份诚实的边界

**它与能力层互补,不与之竞争。** Rulith 决策内核不是 VLA 模型、不是运动控制器、也不是"机器人 OS"。它坐在
任何 agent 或机器人栈的*里面*或*上面*,做**决策/安全内核**——决定一个动作*是否* sound 且被授权的那一层;
学习策略决定*怎么*动,经典反射层让身体物理上安全。三者组合。**Rulith 决策内核是让*决策*可信且可审计的那
一部分。**

**它与运行时监控 / 形式护栏同行的差异**,不在"用了符号"(很多都用),而在它是一个**连贯的内核**:决策
必须被*派生*(而非事后才检查)、一**套统一不变量**(而非某条定制性质)、以及每个结论上的**原生出处**——
正是后者让整套东西可认证。它还*供给*精确推理,不只做安全。

**诚实的边界。** 已*证*的是*机制*:门挡得住、不变量被强制(每条都钉在一个测试上),且保证在沙盒与单次
真机跑上端到端演示过;底层板的与模型无关性在《The Driving Floor》里实证。仍属**经验、待做**的是大规模
真实覆盖:对脏的真实 adapter、失败、并发、长 horizon 的鲁棒;真实任务上驱动的可靠性;以及一次部署所供
*领域模型*的正确性(内核保证的是*从*前提出发的 sound 推理,**不**保证前提为真、也不保证一套手写规则集
完备)。我们把真实任务覆盖当成一个**靠与伙伴部署去回答的问题**,而非从理论里假设——与论文同一条纪律。

## 11. 梯子的下一级:从一生到群体

梯子在"一生"之上还有一级:**群体(fleet)**——很多 agent 共用一部宪法。**联邦治理**把"提议—裁决"推到人口尺度,长出四个单个"一生"没有的群体级对象:任何单个 agent 都改不动群体的法(修宪走群体级授权,绝非个体);一个 agent 的承诺不能与另一个矛盾;一项已验证的能力——领域包、证明——**带着它的信任 floor** 在群体里传播(能力公地,不洗信任);群体不变量是*群里没有任何一个 agent 能无痕越过已建模的法*。

这是 rulith 作为**基础设施**的远端形态:一块共享、受治理的 substrate,把每个 agent 的板、记忆、律包托在上面,像电网或银行系统支撑一个社会。它需要的三条开核接缝——单 agent 隔离、能力公地、共享 solver——**已建**;治理**运行时**(共享宪法修订、跨 agent 一致性、人口级安全)仍是**蓝图**,理论先行、尚未验证。① 仍是脊椎——群体是 ③ 的*联邦化*,不是新内核——诚实底线不变:*已建模的法之内的携证行为*,不是"法一定对"。

## 12. 结语

自主性被采纳的速度,恰好等于它能被*信任*的速度,而信任要求可验证。Rulith 决策内核让信任**结构化**(决策
必须经推导得来)、**统一**(十二条不变量加四条跨会话记忆不变量,精确说出"不失控"是什么意思)、**与模型无关**(对任何 driver 成立)、
且**可认证**(每个决策自带证明)。它底下的推理板是开放且有论文背书的;内核的理论与参考实现也开放接受检验(<https://github.com/rulith-dev/rulith>)——因为对一个全部价值就是*信任*的系统,**开放可查不是让渡价值,而是产品本身。**

我们邀请研究者、建造者、审计者来打它、扩它,并拿它的不变量约束它。
